1. Uvod i Pregled Projekta

1.1 Opis Projekta

BMW Performance Store je moderna web prodavnica delova za BMW vozila, razvijena kao statička web aplikacija koristeći čisti HTML, CSS i JavaScript. Aplikacija omogućava korisnicima da pretražuju, filtriraju i kupuju originalne BMW delove sa VIN verifikacijom kompatibilnosti.

1.2 Ključne Karakteristike

  • VIN Dekodiranje: Automatsko prepoznavanje modela vozila iz VIN broja
  • Napredno Filtriranje: Filtri po modelu, kategoriji, brzi filteri i pretraga
  • Dinamička Korpa: Ažuriranje korpe u realnom vremenu sa lokalnim skladištenjem
  • Proces Naplate: Kompletan formular za poručivanje sa validacijom
  • Responzivni Dizajn: Optimizovano za sve uređaje
  • Moderna UI: Tamna tema sa gradijentima i animacijama

1.3 Podržani Modeli

Aplikacija podržava sledeće BMW modele:

  • E28 (5 serija, 1981-1988)
  • E30 (3 serija, 1982-1994)
  • E46 (3 serija, 1998-2006)
  • E60 (5 serija, 2003-2010)
  • E90 (3 serija sedan, 2005-2012)
  • E91 (3 serija touring, 2005-2012)
  • E92 (3 serija coupe, 2006-2013)
  • F80 (M3, 2014-2018)
  • F87 (M2, 2015-2021)
  • G80 (M3, 2021+)

4.1.2 Filtriranje po Kategoriji

5.5 VIN Struktura

VIN (Vehicle Identification Number) se sastoji od 17 karaktera:

  • Pozicije 1-3 (WMI): World Manufacturer Identifier (npr. WBA za BMW)
  • Pozicije 4-5: Model kod (npr. VF za 3 serija sedan)
  • Pozicija 6: Tip vozila (1=sedan, 3=sedan, 5=touring, itd.)
  • Pozicija 10: Godina proizvodnje
  • Pozicije 11-17: Serijski broj

6. VIN Dekodiranje

6.1 Pregled Funkcionalnosti

VIN dekodiranje je jedna od ključnih funkcionalnosti aplikacije. Omogućava automatsko prepoznavanje modela vozila iz VIN broja i automatsko filtriranje proizvoda.

6.2 Validacija VIN Formata

Funkcija validateVinFormat() proverava:

  • Dužinu VIN broja (mora biti tačno 17 karaktera)
  • Dozvoljene karaktere (samo alfanumerički, bez I, O, Q)
  • Format validnosti

6.3 Parsiranje VIN Strukture

Funkcija parseVinStructure() izdvaja:

  • WMI (pozicije 1-3)
  • Model kod (pozicije 4-5)
  • Tip vozila (pozicija 6)
  • Godinu proizvodnje (pozicija 10)

6.4 Detekcija Modela

Funkcija detectModelFromVin() koristi prioritetni sistem:

Prioritet 1: Strukturno prepoznavanje sa model kodom

Najpreciznije - kombinuje WMI + model kod + poziciju 6 + godinu

Primer: WBA + VF + pozicija 6 (1 ili 3) + godina (2005-2012) = E90

Prioritet 2: Specifični pattern matching

Regex patterni za specifične kombinacije (npr. VF kod za E90)

Prioritet 3: Strukturno prepoznavanje bez model koda

WMI + pozicija 6 + godina proizvodnje

Prioritet 4: Opšti pattern matching

Najniži prioritet - opšti regex patterni

6.5 VIN Model Mapa

vinModelMap objekat sadrži pravila za svaki model:

{
  E90: [
    { wmi: 'WBA', modelCode: 'VF', years: [2005, 2012] },
    { pattern: /^WBAVF[0-9A-Z]{5}/ },
    ...
  ],
  E30: [
    { wmi: 'WBA', pos6: ['1', '3'], years: [1982, 1994] },
    { pattern: /^WBA(?!VF)[A-Z0-9]{2}[A-Z0-9]{2}[13][0-9]{2}/ },
    ...
  ],
  ...
}

6.6 Integracija sa Bimmer.work

Aplikacija pokušava da koristi bimmer.work API za dekodiranje:

  • Pokušava direktnu integraciju (može imati CORS probleme)
  • Parsira JSON ili HTML odgovor
  • Izdvaja model iz odgovora
  • Rezervna opcija na lokalno dekodiranje ako API ne radi
  • Dugme za otvaranje bimmer.work sajta sa VIN brojem

6.7 Automatsko Postavljanje Filtera

Nakon uspešnog dekodiranja VIN broja:

  • Postavlja se state.selectedModel
  • Ažuriraju se filteri za modele (buildModelFilters())
  • Ažuriraju se kategorije (buildCategories())
  • Renderuju se proizvodi samo za prepoznati model
  • Prikazuje se poruka sa informacijama o modelu

7. Korpa i Proces Naplate

7.1 Funkcionalnosti Korpe

7.1.1 Dodavanje u Korpu

Funkcija addToCart(productId):

  • Proverava da li proizvod već postoji u korpi
  • Ako postoji, povećava količinu
  • Ako ne postoji, dodaje novu stavku
  • Ažurira localStorage
  • Prikazuje korpu ponovo

7.1.2 Promena Količine

Funkcija changeQuantity(productId, delta):

  • Povećava ili smanjuje količinu za određenu vrednost
  • Proverava da li je količina veća od 0
  • Ako je količina 0, uklanja stavku
  • Ažurira localStorage i prikazuje korpu

7.1.3 Uklanjanje iz Korpe

Funkcija removeFromCart(productId):

  • Uklanja stavku iz korpe
  • Ažurira localStorage
  • Prikazuje korpu ponovo

7.1.4 Renderovanje Korpe

Funkcija renderCart():

  • Učitava korpu iz localStorage
  • Za svaku stavku učitava podatke proizvoda
  • Kalkuliše ukupnu cenu
  • Prikazuje broj stavki u korpi
  • Aktivira/Deaktivira dugme za naplatu

7.2 Stranica za Naplatu

7.2.1 Struktura Forme

Forma za naplatu sadrži sledeće sekcije:

  • Lični podaci: Ime, Prezime, Email, Telefon
  • Adresa isporuke: Ulica, Grad, Poštanski broj, Država
  • Način plaćanja: Gotovina, Kartica, Bankovni transfer
  • Napomena: Opcioni tekst za dodatne instrukcije

7.2.2 Validacija

  • HTML5 validacija za obavezna polja
  • Validacija formata emaila
  • Provera da korpa nije prazna

7.2.3 Proces Slanja Porudžbine

Funkcija handleSubmit():

  1. Preuzima podatke iz forme
  2. Kreira objekat porudžbine sa svim podacima
  3. Dodaje vremensku oznaku
  4. Čuva porudžbinu u localStorage pod ključem 'bmwOrders'
  5. Briše korpu iz localStorage
  6. Prikazuje poruku o uspešnoj porudžbini
  7. Preusmerava na glavnu stranicu

7.2.4 Pregled Porudžbine

Stranica za naplatu prikazuje:

  • Listu stavki iz korpe sa količinama
  • Međuzbir (bez dostave)
  • Cenu dostave (€15.00)
  • Ukupnu cenu
  • Informacije o isporuci i garanciji

9. Responzivni Dizajn

9.1 Strategija Prelomnih Tačaka

Prelomna Tačka Širina Primena
Mobilni < 768px Jedna kolona, vertikalni raspored, mobilni meni
Tablet 768px - 1024px Dve kolone, prilagođena mreža
Desktop > 1024px Puna širina, mrežni raspored, fiksna bočna traka

9.2 Responzivne Komponente

9.2.1 Glavna Sekcija

  • Mobilni: Vertikalni raspored, kartice jedna ispod druge
  • Desktop: Horizontalni raspored, jedna pored druge

9.2.2 Filteri

  • Mobilni: Puna širina, filter grupe jedna ispod druge
  • Desktop: Organizovani u karticu filtera sa grupama

9.2.3 Mreža Proizvoda

  • Mobilni: 1 kolona
  • Tablet: 2 kolone
  • Desktop: 3-4 kolone (automatsko popunjavanje sa minmax)

9.2.4 Korpa

  • Mobilni: Puna širina, na dnu stranice
  • Desktop: Fiksna bočna traka, 320px širine

9.2.5 Naplata

  • Mobilni: Vertikalni raspored, forma ispod pregleda
  • Desktop: Jedna pored druge, forma levo, pregled desno
